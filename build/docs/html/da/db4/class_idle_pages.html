Detailed Description

Handles story book pages for IdleTTD.

Renders and displays IdleTTS screens and keeps track of rendered elements (buttons) for click handling.

◆ ShowIdleReportScreen()

function IdlePages::ShowIdleReportScreen ( int idleBalance = 0,
int inactiveSeconds = 0,
SummaryVehicleStats vehicleSummary = null,
std::array< SummaryVehicleStats > allVehicleStats = null,
int intLastYearBalance = 0 )

Shows idle report screen.

Clears idle story page, renders idle report elements with this.RenderIdleReportScreen() and shows the page to the player. For details on idle report contents check out RenderIdleReportScreen function.

Note
This will set IdleReportVisible flag to true, causing MainClass.Start() while loop to run a lot more often than normal. It might slow the game down but ensures that script react promptly to clicks and other player actions.
Attention
Idle report page is quasi-persistent. If player closes story book without clicking the button within, script will open idle report story book page again and keep doing it until player clicks the button on the page.
Parameters
idleBalanceIdle balance to use
inactiveSecondsInactive idle seconds
vehicleSummarySummary of all vehicles
allVehicleStatsAn array of vehicle stats for all vehicle types
intLastYearBalanceLast year balance for calculations
Returns
bool
Return values
trueIdle report story book page is displayed
falseFailed opening idle report story book page

◆ IdleReportVisible

IdlePages::IdleReportVisible = false

Idle report visibility flag.

If user action (clicking the button on idle report story book page) is required this will be set to true

Attention
Setting this to true will result in script taking as much resources as it can, possibly hampering the perfrmance.

◆ IntroVisible

IdlePages::IntroVisible = false

Intro visibility flag.

Set to true to speed up handling button click (only if idle story book page is also visible).

Note
If story book window is open true value here will result in script taking as much resources as it can, possibly hampering the perfrmance. Closing the story book window (without clicking on the button within) returns script performance to normal but won't reset the value to false.
